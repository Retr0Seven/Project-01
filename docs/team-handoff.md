# Team Handoff

Use this file to split real remaining work between teammates.

## Recommended Teammate Tasks

- Fill out `docs/manual-test-report.md` after testing the app locally.
- Render `docs/class-diagram.puml` and `docs/deployment-diagram.puml` to PNG or PDF.
- Record the 3-minute demo using `docs/demo-checklist.md`.
- Add screenshots of the server and client windows to the README or submission package.
- Review the source code and open small cleanup commits if anything is unclear.
- Confirm the final submission against `docs/submission-checklist.md`.

## Honest Contribution Rule

Each teammate should commit their own changes from their own Git account. Use `Co-authored-by:` only when both people actually worked on the same commit.

Example:

```text
Co-authored-by: Teammate Name <teammate@email.com>
```

Do not change commit authorship to credit someone for work they did not do.
